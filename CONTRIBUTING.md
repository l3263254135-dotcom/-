# 贡献指南

感谢您对每日餐食助手项目的兴趣！这份指南将帮助您了解如何为项目做出贡献。

## 如何贡献

### 1. Fork 和 Clone

\`\`\`bash
# Fork 项目
# Clone 您的 fork
git clone https://github.com/your-username/daily-meal-assistant.git
cd daily-meal-assistant
\`\`\`

### 2. 创建特性分支

\`\`\`bash
git checkout -b feature/your-feature-name
\`\`\`

### 3. 进行更改

请确保您的代码遵循项目的编码风格。

### 4. 测试

\`\`\`bash
npm test
npm run lint
\`\`\`

### 5. 提交变更

\`\`\`bash
git commit -m "Add feature: description of your changes"
\`\`\`

### 6. Push 到您的 Fork

\`\`\`bash
git push origin feature/your-feature-name
\`\`\`

### 7. 创建 Pull Request

在 GitHub 上创建一个 Pull Request，并详细描述您的更改。

## 报告 Bug

如果您发现了一个 bug，请：

1. 使用 GitHub Issues 创建一个新的 issue
2. 提供清晰的标题和描述
3. 包含复现问题的步骤
4. 附上预期行为和实际行为

## 建议新功能

有新功能的想法？请：

1. 创建一个 GitHub Issue
2. 描述功能的用途和好处
3. 提供可能的实现方案

## 代码规范

- 使用 TypeScript
- 遵循 ESLint 规则
- 写出清晰的注释
- 保持代码简洁易读

## 许可证

通过贡献代码，您同意将您的代码许可证设置为 MIT。

感谢您的贡献！🎉
